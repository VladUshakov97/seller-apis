# seller.py — скрипт синхронизации остатков и цен с Ozon

### Назначение
Скрипт автоматически обновляет информацию о ценах и остатках товаров в интернет-магазине Ozon, используя данные с сайта `timeworld.ru`.

### Что делает скрипт
1. Загружает актуальный файл с остатками товаров с внешнего сайта (Casio).
2. Получает список товаров из вашего магазина на Ozon.
3. Сравнивает внешние данные с тем, что уже загружено в Ozon.
4. Обновляет остатки на складе Ozon
5. Обновляет цены на Ozon в соответствии с файлом (конвертируя цену из формата `5'990.00 руб.` в `5990`).
6. Удаляет временный файл после обработки.


# market.py — скрипт для управления остатками и ценами на Яндекс Маркете

## Что умеет скрипт

Этот скрипт автоматически обновляет информацию о товарах в магазинах на платформе **Яндекс Маркет**.

### Основные возможности

- **Загрузка остатков** — получает актуальные данные о количестве товаров из локального файла (например, из Excel) и обновляет их на складах.
- **Обновление цен** — синхронизирует цены на товары с теми, что указаны в загруженных данных.
- **Поддержка двух типов поставок** — работает одновременно с двумя моделями: FBS (Fulfillment by Seller) и DBS (Delivery by Seller). Для каждого типа используется свой склад и кампания.
- **Разбивка на пакеты** — при отправке большого количества товаров автоматически разбивает данные на части, чтобы не превышать лимиты API Яндекс Маркета.
- **Обработка отсутствующих товаров** — если товар есть в базе Маркета, но не найден в локальном файле, его остаток устанавливается в 0.

### Что происходит при запуске

1. Скрипт загружает текущий список остатков и цен из вашего источника.
2. Получает с Маркета список всех товаров, которые уже загружены в витрину.
3. Сравнивает данные и формирует новые значения остатков и цен.
4. Отправляет обновления на Маркет отдельно для FBS и DBS.
5. В случае ошибок соединения или таймаутов выводит понятные сообщения.

### Для чего это нужно

Скрипт позволяет продавцам на Яндекс Маркете автоматизировать рутинные задачи — не обновлять вручную цены и количество товаров каждый раз, когда что-то продаётся или меняется в учётной системе. Достаточно один раз запустить скрипт, и все изменения попадут в витрину.
